Setting up Vizi

Last updated: 1st Oct, 2017

Welcome

Vizi is a platform designed for schools to easily collate, report and draw insight from performance data. It allows teachers to truly understand their students through data. Vizi provides the transparency and accuracy to identify both students who excel, and those who may be struggling, allowing support to be provided in the crucial early stages.

The admin tools section is restricted to teachers with the admin permissions linked to their profile. Admin permissions are controlled by the 'Vizi Champion' who can add and remove admin permission to teachers.

Setup

Users

Sidebar - (Admin Tools)(Users)

What is a user?

In Vizi, a user can be a teacher, student or guardian. These different user types restrict the users access and permissions within Vizi.

Teachers can create edit and view performance data for the groups they are associated to (groups and groups types). Teachers can also have view read only access to collections of groups that they are associated to (Collections). This allows a teacher to view all Physics groups if they were Head of Physics, but restrict their ability to create and edit performance data. Teachers can have administration rights, this is designed to restricted to a select few IT staff and allows the user to manage the setup and parameters inn Vizi.

A student user is restricted to viewing data relating to themselves only. A student can view their marks, upcoming assessments and messages a teacher has shared with that user or a user's group.

A guardian user is restricted to viewing data relating to their student or students only.

N.B - For single user accounts there can only be a single teacher user type. This user will have full administration rights.

Steps

Create

To create a user, click on the create button to get directed to the the create user page. User details that are required include the following:
- Name
- User type (student, teacher, parent)
- Email address
- Whether the user will require a login
Click on the create button to confirm the creation of a new user. If the login required checkbox is checked, the user will be sent login details and can start using their account. An email address for the corresponding user must be completed for the user to receive a login.

Manage

To edit or delete users, select a user and click the edit button. Here a user can update user details or delete the user. Users can add performance metrics to student accounts by clicking the 'Update performance metrics' button. Here users can add grades for each performance metric in Vizi and for each of the student's subjects.

Deletions

When a user is deleted, the user will no longer appear in Vizi and that user's data will not contribute to the performance data displayed throughout Vizi. User data will be archived and remain recoverable for three months post deletion. Following the three month archive period all user specific data will be permanently deleted.

Dependancies

Users are not dependant on any other aspects of Vizi setup, however, to add performance metrics to student accounts relies on the creation of Grade types, Grades and Performance metrics

Group types

Sidebar - (Admin Tools)(Group Types)

What is a user?

A group is an aggregation of users. There are two groups that come preprogammed into Vizi; Sets and Forms. These can be renamed or deleted and custom groups added to suit an educational institution.

Steps

Create

To create a new group type click the 'Create' button and enter the name of the new group type. Click the 'Create group type' button to cofirm the creation of a new group. Once a group type is created, a user can create groups of that group type. In the example of sets, a user would be able create sets '7A', '7B', '7C' etc....

Manage

To edit or delete a group type, select a group type and click the edit button. A user can rename, delete and create custom settings for different group types. These include the following:
- Create group type owner. In most cases a member of IT staff would be the owner.
- Has subject(s) - does the group type need subjects associated to it. For example, you may have subjects associated to 'Sets', but not for 'Forms'.
- Has markbook - does the group type need to have marks recorded for it. For example, you may have want to record marks for both 'Sets' and 'Forms'.
- View permissions [Unfinished.]

To save the group type details, click on the 'Update group type' button.

Deletions

If a group type has been created, but has not been used throughout Vizi, it can be permanently deleted and will not appear in the group types table. If the group type has already been used throughout Vizi, it can not be retrospectively deleted, rather, there will be an option to disable the group type from future use. This will mean historic data relating to that group type will remain, but it can not be used going forward. Note - rather than deleting and creating a new group type, it is often mroe suitable to rename and edit existing group types.

Groups

Sidebar - (Admin Tools)(Groups)

What are Groups?

Once a group type has been created, that group type will appear in under the groups menu, in admin tool in the sidebar. Groups can be created for your group types. For example, for the group type 'Sets' a user can create sets '7A', '7B', '7C' etc....

Steps

Create

To create a group, select the desired group type under the groups submenu. Click the 'Create' button to begin creating a group.

Enter the group name, the teacher(s) allocated to that group. Users will be able to add subjects to that group if that 'Has subject(s)' checkbox has been selected when creating the group type. Click on the 'Create group' button to create the group.

To add or remove students to or from that group select the add/remove students button. Use the add buttons next to a student's name to add students to the group and the subtract buttons to remove them. A user can search for student using the search box. Once all the desired students have been added to the group, select the update group button to confirm the selection.

Manage

To edit or delete a group, select the a group and click the 'Edit' button. Modify the group name, the teacher or teachers and add or remove students from that group. Click the update group button to update the group the group.

Deletions

If a group has been created, but has not been used throughout Vizi, it can be permanently deleted and will not appear in the groups table. If the group type has already been used throughout Vizi, it can not be retrospectively deleted, rather, there will be an option to disable the group type from future use. This will mean historic data relating to that group will remain, but it can not be used going forward. Note - rather than deleting and creating a new group, it is often mroe suitable to rename and edit existing group types.

Dependancies

Creating groups relies on the creation of Group types, Subjects.

Year end

Groups will be archived at the end of each academic year. Archived groups will no longer appear on a user's sidebar, but data relating to archived groups can still be retrieved and integrated with current performance data. Groups with the existing settings and users can be resued for the next academic year by updating the academic year for that group (this will not affect the archive process). [clarify process]

Assessment States

Sidebar - (Admin Tools)(Miscellaneous)(Assessment states)

What is?

Assessment states are used to define the completion status of an assessment for individual students and subsequently dictate the completion status for the overall assessment. For example, assessment states might be 'Late', 'On time' or 'Exempt'.

For each assessment state the user will define whether the state requires completed grades for it to be deemed complete. For example, for the status 'On-time' a user might select 'Grades required for completion' and for the status 'Exempt' they will not. This will mean grades must be entered for the students with the status 'On time' for the assessment to be deemed completed. For those students marked exempt, no grades must be entered for the assessment to be marked as complete.

Assessment states help create key performance metrics, such as % assessments complete and assessment outstanding.

Steps

Create

New assessment states can be created by clicking the create button and entering the name of the assessment state. The user should select the 'Grades required for completion?' checkbox if students marked with this state require a grade to be assigned to them for the assessment to be deemed as complete.

Manage

Existing assessment states can be managed by selecting a state in the table and clicking the edit button. Assessment state names can be changed, their requirement for grades for completion status altered or they can be deleted.

Deletions

If a assessment state has been created, but has not been used throughout Vizi, it can be permanently deleted and will not appear in the assessment states table. If the assessment state has already been used throughout Vizi, it can not be retrospectively deleted, rather, there will be an option to disable the assessment state from future use. Note - rather than deleting and creating a new assessment state, it is often more suitable to rename and edit the existing assessment state.

Grade Types

Sidebar - (Admin Tools)(Miscellaneous)(Grade Types)

What is?

Different grading systems can be created and edited to suit a user or school. Multiple grade types can be created and performance data relating to mulitple grade types can be recorded for each assessment.

Vizi comes with the grade type 'Attainment' preprogammed. This grade type is designed to measure the level of academic performance of a student. Grade types can be renamed or deleted as per the user's or school's requriements.

Grades are created for eachgrade type. In the case of the preprogrammed 'Attainment' grade type, grades 'A' through to 'F' have been created for that grade type.

Steps

Create

To create a grade type, click the 'Create' button and enter the grade type name, a description and select whether this grade type will be the default grade. By selecting the grade type as the default grade, it will always appear as a grade type when setting an assessment (this can be removed and alternate grade types added per a user's requirements).

Manage

Grade types can be edited, deleted and have grades created for by selecting the grade type from the grade types table and clicking the 'Edit' button.

Grades can be added to the grade type by clicking the 'Add Grade' button at the top of the grades table. Enter the name of the grade and click the 'Create' button. This will add the grade to the grade table. Once all grades for a grade type have been created a user can order the grades by selecting the up/down arrows in the rank column. Grades at the top of the list should represent the highest marks and at the bottom of the list, the lowest. Users can edit and delete the grades by selecting the icons in the respective columns.

For each grade a numerical value is assigned to the grade to allow comparison of different grade types across vizi. These values are between 0-100. Grade values and boundries are assigned automatically based on the rank and the number of grades for a grade type. Performance metrics

Sidebar - (Admin Tools)(Miscellaneous)(Grade Types)

What is?

Performance metrics are created using grade types and allow the allocation of specific grades to students for comparison against their actual peformance. For example, the performance metric 'Minimun attainment target' could be created based on the 'Attainment' grade type. Or the peformance metrics 'Target application' based on the 'Application' grade type. Each student can be alloacted a grade for these performance metrics for each of their subjects. For example, Joe Bloggs may be alloacted an 'A' for the performanc metric 'Minimun attainment target' in Mathematics. This can then be used throughout Vizi to highlight grades that vary from that performance metric.

Steps

Create

Click the create button to create a new performance metric. Enter the name for the performance metric, a brief description and select the grade type the performance metric will be based on.

Manage

To manage peformance metrics, select a performance metric and click the edit button. A user can edit the name of the performance metric, the description and and the grade type the performance metric will use. If a user changes the name or description of the performance metric, the performance metric will update across Vizi and individual student's performance metrics will not be lost. If a user changes the grade type to which the performance metric refers, all student's performance metric grades for that performance metric will be lost.

Deletions

Deleting a performance metric will result in the the grades assigned to students for that performance metric being deleted. The performance metric will no longer appear in performance data for comparison.

Dependancies

Peformance metrics are dependant on the creation of grade types and grades .

Calendars

Sidebar - (Admin Tools)(Calendars)

What is?

The Vizi calendar can be used to manage complete timetables or to simply remind users of adhoc events. By creating events in Vizi, users will get reminders of upcoming events and have the ability attribute performance data to specific events, adding depth to the data. Events can be added to Vizi for a groups or collections and will appear in the calendars for all those users with those groups or collections.

Steps

Create

A user can select the group type for which they want to create an event and then select the specific group they want to create the event for. A user can then add the following details to an event:

- The date - the date for which the event will occur.
- The time - the time at which the event will occur.
- Whether the event is recurring, which includes the following parameters:

  • Restrict the event to term time (a user must have entered term times in the calendar settings page).
  • Repeat every x number of weeks. For example, repeat every 2 weeks will enter the event fortnightly.
  • The days in the week a user would like the event to occur. If this selection conflicts with the initial date (for example, the initial date is a Tuesday, but a user has specified the event repeat every Wednesday), the initial date will be ignored.
  • When the event series will end. This can be at the end of the academic year, after # appearances or on a specific date.
- The event title - the title that will appear on user's calendar for a particular event.
- The event type - the type of event an event is (a user must have entered event types in the calendar seetings page). Different event types can have different permissions. For example, allowing the user to record marks for some event types but not others.
- The subject.
- The groups and the users that are invited to the event. This will make the event appear on the specified groups and users calendar.

Manage

Manage an event by filtering for the desired group type and group and then using the calendar display, click on the event. This gives the user the option to edit or delete the event.

For an edit or deletion the user will have the option to edit or delete the event for:
- That event only - this will only make changes to the event a user selected.
- That event and all subsequent events in the series - this will make changes the event a user selected and all subsequent events in that series.
- All events in the series; this will make changes to the complete series of events.

Deletions

Deleting an event will not impact any peformance data relating to that event or series of events. The event will no longer appear in users' calendars.

Terms define the boundries of the term times within the academic year. Term times will effect the calendar function across Vizi, providing boundries for recurring events. Terms are attached to academic years, so new terms should be created as new years come around.

Event Types

Sidebar - (Admin Tools)(Miscellaneous)(Event Types)

What is?

Event types allow the user to assign different properties to events in the Vizi calendar. When an event is created in the calendar, a user will select event type. Examples of event types are 'Lesson', Registration', 'Lunch break' etc....

A user can add the following properties to an event type:

- Can take assessments - This will allow the user to select an event in the calendar and set an assessment for that event.
- Can take registers - This will allow the user to select an event in the calendar and take a register for that event.
- Events can be created for [List of your groups] - This will allow the user to create this event for specific group types only. For example, the user may want to restrict the event type 'Form registration' to the group type 'Forms' only.


For example, a user might create 'Lesson' with the capability to take set an assessment and take a register. Or create 'Registration' with the capability just to take a register.

Steps

Create

Click the create button to create an event type, enter the event type name and select properties for that event type.

Manage

Edit the name of the event type and select whether users canset assessments and take registers for that event type. Select which groups can be added to this event type in the calendar.
